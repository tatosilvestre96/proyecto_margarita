# 📋 Categorías de Backlog - Referencia Rápida

## 🏗️ Backend
Código: `backend`  
Para: APIs, endpoints, lógica de servidor, autenticación

**Ejemplos:**
- Crear endpoint para guardar progreso
- Implementar validación de ubicación
- Optimizar queries de base de datos

---

## 🎨 Frontend
Código: `frontend`  
Para: UI, componentes React, estilos, animaciones

**Ejemplos:**
- Mejorar responsive del mapa
- Diseñar pantalla de resultados
- Crear componente de login

---

## 🗺️ Gameplay
Código: `gameplay` (o `game`, `juego`)  
Para: Mecánicas, acertijos, misiones, lógica del juego

**Ejemplos:**
- Crear acertijo para Teatro Colón
- Sistema de hints para acertijos
- Modo cooperativo

---

## 🎬 Video Content
Código: `video` (o `contenido`, `videos`)  
Para: Videos, narrativa, cinematografía, efectos

**Ejemplos:**
- Grabar intro del primer nivel
- Editar video del Teatro Colón
- Música de fondo

---

## 🗄️ Database
Código: `database` (o `db`, `datos`)  
Para: Schema, migraciones, queries, optimizaciones

**Ejemplos:**
- Agregar índices a tabla de usuarios
- Crear backup strategy
- Optimizar queries lentas

---

## 🔐 Seguridad & Performance
Código: `security` (o `seguridad`, `performance`)  
Para: Seguridad, autenticación, performance, escalabilidad

**Ejemplos:**
- Agregar rate limiting
- Implementar caching
- Validar entradas del usuario

---

## 📱 Multiplayer
Código: `multiplayer` (o `multi`, `collab`)  
Para: Modo colaborativo, rivalidad, social, real-time

**Ejemplos:**
- Sincronización en tiempo real
- Modo rivalidad con leaderboard
- Sistema de invitaciones

---

## 🐛 Bugs & Fixes
Código: `bug` (o `bugs`, `fix`)  
Para: Problemas encontrados, correcciones urgentes

**Ejemplos:**
- Socket.io desconexión después de 5 min
- Token refresh no funciona
- Mapa no carga en móvil

---

## 📚 Documentación
Código: `docs` (o `documentación`, `doc`)  
Para: Documentación, guías, tutoriales, comentarios en código

**Ejemplos:**
- Documentar API endpoints
- Crear guía de instalación
- Comentar funciones complejas

---

## 🎯 Mejoras Generales
Código: ninguno (por defecto)  
Para: Ideas generales, mejoras, refactoring, technical debt

**Ejemplos:**
- Crear versión MVP
- Refactorizar componentes
- Limpiar código antiguo

---
